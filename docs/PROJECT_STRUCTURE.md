# PROJECT_STRUCTURE.md

## 1. Назначение документа
Этот документ описывает стандартизированную структуру каталогов и файлов проекта MarlaisSMP Core. Строгое следование этой структуре обязательно для всех новых модулей и компонентов системы.

## 2. Глобальная структура репозитория

```text
MarlaisSMP-Core/
├── .gitignore               # Исключения для Git
├── build.gradle.kts         # Главный скрипт сборки Gradle (Kotlin DSL)
├── settings.gradle.kts      # Настройки мультимодульной сборки (если потребуется)
├── README.md                # Главная страница проекта на GitHub
│
├── docs/                    # Вся проектная документация
│   ├── AGENTS.md            # Роли и правила работы AI
│   ├── PROJECT_STRUCTURE.md # Этот файл
│   └── specification/       # Технические задания (ТЗ) на компоненты
│       ├── 00_PROJECT_OVERVIEW.md
│       ├── 01_SYSTEM_ARCHITECTURE.md
│       └── 02_DATA_MODEL.md
│
└── src/                     # Исходный код проекта
    ├── main/java/           # Основной код ядра
    ├── main/resources/      # Ресурсы (конфиги по умолчанию, манифесты)
    ├── test/java/           # Unit и интеграционные тесты
    └── test/resources/      # Ресурсы для тестов
```

## 3. Структура Java-пакетов (src/main/java/net/marlais/core/)

Весь код ядра должен находиться в базовом пакете `net.marlais.core`. Внутри него структура разделяется по зонам ответственности:

* `model/` — DTO, интерфейсы данных (например, `CorePlayer`, статусы).
* `service/` — Службы ядра и контейнер (например, `ServiceContainer`, `StorageService`).
* `event/` — Шина событий и сами события.
* `exception/` — Кастомные ошибки (например, `ServiceNotFoundException`).
* `config/` — Классы для загрузки и хранения конфигураций.
* `module/` — Загрузчик внешних аддонов и API для них.

## 4. Правила размещения
- Никакой логики не должно быть в корневом пакете `net.marlais.core` (кроме главного класса плагина, если он необходим).
- Тесты в `src/test/java/` должны полностью зеркалировать структуру пакетов из `src/main/java/`.
