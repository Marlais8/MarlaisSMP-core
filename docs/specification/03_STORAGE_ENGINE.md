# 03_STORAGE_ENGINE.md

## 1. Назначение документа
Этот документ описывает архитектуру и API движка хранения данных (Storage Engine) для MarlaisSMP Core. Движок отвечает за сохранение и загрузку профилей игроков (`CorePlayer`) и конфигураций модулей.

## 2. Архитектурные требования
- **Асинхронность (Asynchronous by default):** Все операции ввода-вывода (I/O) при работе с диском должны выполняться в отдельных потоках, чтобы не блокировать основной игровой поток сервера.
- **Отказоустойчивость (Atomicity):** Запись файлов должна происходить по принципу "сначала пишем во временный файл, затем заменяем старый". Это предотвратит потерю данных игрока, если в момент сохранения выключится свет или упадет сервер.
- **Потокобезопасность:** Доступ к кэшу данных в оперативной памяти должен быть строго синхронизирован.

## 3. Интерфейс StorageService

Служба хранения регистрируется в нашем `ServiceContainer` и предоставляет другим модулям простой API.

```java
package net.marlais.core.service;

import net.marlais.core.model.CorePlayer;
import java.util.UUID;
import java.util.concurrent.CompletableFuture;

public interface StorageService extends Service {
    /**
     * Асинхронно загружает профиль игрока из хранилища.
     * Если игрока нет, возвращает пустой/новый профиль.
     */
    CompletableFuture<CorePlayer> loadPlayer(UUID uniqueId);

    /**
     * Асинхронно сохраняет профиль игрока в хранилище.
     */
    CompletableFuture<Void> savePlayer(CorePlayer player);
}
```

## 4. Формат хранения данных (JSON бэкенд)
Для первой версии ядра мы используем файловую JSON-базу данных:
- Все данные игроков сохраняются в папку `data/players/`.
- Имя файла соответствует UUID игрока (например, `data/players/123e4567-e89b-12d3-a456-426614174000.json`).
- Для сериализации Java-объекта в JSON используется стандартный сериализатор.
