# 00_PROJECT_OVERVIEW.md

## 1. Назначение системы
**MarlaisSMP Core** — это фундаментальное ядро (фреймворк), предоставляющее базовые API, системы хранения данных, управления событиями и загрузки модулей. Ядро само по себе не содержит узкоспециализированной игровой логики, но обеспечивает надежный, быстрый и расширяемый фундамент для всех будущих аддонов сервера.

## 2. Основные архитектурные цели
- **Модульность:** Все внутренние подсистемы и будущие аддоны должны быть максимально изолированы друг от друга.
- **Расширяемость:** Внешние плагины должны иметь возможность подключаться к ядру и изменять его поведение только через четко определенный Public API.
- **Производительность:** Минимальный overhead на Event Bus и Storage Engine. Оптимизация горячих путей (hot paths).
- **Отказоустойчивость:** Сбой в одном второстепенном модуле не должен приводить к падению всего ядра.

## 3. Высокоуровневая концепция
Проект строится на основе **Service-Oriented Architecture (SOA)** в миниатюре:
- Используется паттерн **Service Container** (или Dependency Injection).
- Взаимодействие между модулями происходит через **Event Bus** (слабая связность).
- Доступ к данным абстрагирован через **Storage Engine**, что позволяет легко менять бэкенд базы данных (JSON, SQLite, MySQL и т.д.) без переписывания логики.

## 4. Границы ответственности (Scope)

**Входит в MarlaisSMP Core:**
- Управление жизненным циклом сервисов.
- Шина событий (Event Bus).
- Конфигурация и локализация.
- Абстракция базы данных.
- Система привилегий и базовых команд.
- Модульный загрузчик.

**НЕ входит в MarlaisSMP Core (реализуется аддонами):**
- Специфичные игровые механики (кастомные предметы, мобы, боссы).
- Сложные экономические балансы (только API для экономики).
- Конкретные мини-игры или квесты.

## 5. Глоссарий
- **Core (Ядро):** Базовый плагин, предоставляющий API и системные сервисы.
- **Module/Addon:** Внешний компонент, реализующий игровую логику за счет использования Core API.
- **Service:** Синглтон-класс внутри ядра, инкапсулирующий логику одной подсистемы (например, `StorageService`).
- **Registry:** Место регистрации игровых элементов, команд, пермиссий.
