# 02_DATA_MODEL.md

## 1. Назначение документа
Этот документ описывает единую модель данных MarlaisSMP Core. Он определяет структуры данных для хранения информации об игроках, конфигурациях и состояниях модулей, а также устанавливает правила их сериализации и десериализации.

## 2. Архитектурные принципы работы с данными
- **Иммутабельность (Immutable by default):** Объекты данных (Data Transfer Objects — DTO) должны быть неизменяемыми после создания, чтобы исключить баги при многопоточном доступе. Для изменения данных должен использоваться паттерн Builder или создание копии объекта с новыми значениями.
- **Агностицизм к хранилищу:** Модель данных не должна знать, куда она сохраняется (в JSON-файл, SQLite или MySQL). Задача модели — предоставить чистую структуру.
- **Идентификация по UUID:** Все игроки и сущности идентифицируются исключительно по UUID (Universally Unique Identifier), а не по никнеймам.

## 3. Базовые сущности ядра

### 3.1. Игровая сессия и профиль игрока (CorePlayer)
Базовый объект, представляющий данные игрока на сервере.

```java
package net.marlais.core.model;

import java.util.UUID;
import java.util.Map;
import java.util.Optional;

/**
 * Представляет профиль игрока в системе MarlaisSMP Core.
 */
public interface CorePlayer {
    /**
     * Уникальный UUID игрока.
     */
    UUID getUniqueId();

    /**
     * Последний известный никнейм игрока.
     */
    String getLastKnownName();

    /**
     * Время первого входа на сервер (timestamp в миллисекундах).
     */
    long getFirstPlay();

    /**
     * Время последнего входа на сервер.
     */
    long getLastPlay();

    /**
     * Получить метаданные, сохраненные сторонними модулями.
     */
    Map<String, Object> getMetadata();

    /**
     * Безопасное получение метаданных по ключу.
     */
    Optional<Object> getMetadataValue(String key);
}
